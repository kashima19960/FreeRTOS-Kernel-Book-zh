# 11 低功耗支持

## 11.1 省电简介

FreeRTOS 通过空闲任务钩子与无滴答（Tickless）空闲模式，为低功耗模式提供了便捷入口。

通常使用空闲任务钩子将运行 FreeRTOS 的微控制器置于低功耗状态来降低功耗。
这种方式的节能效果会受到定期 tick 中断的限制：必须周期性退出并重新进入低功耗状态
以处理 tick 中断。此外，若 tick 中断频率过高（唤醒过于频繁），每个 tick 进入/退出
低功耗状态所消耗的时间与能量将抵消节能收益，除非使用极轻量的省电模式。

FreeRTOS 支持一种允许微控制器在低功耗状态间周期性进出的机制。Tickless idle 模式
会在空闲期间（无可运行应用任务）停止周期 tick 中断，使 MCU 能保持在深度省电状态，
直到发生中断或需要将某任务切换为 *Ready* 状态。随后在重启 tick 中断时，对 RTOS
tick 计数进行修正。Tickless 模式的原则是：当 MCU 运行空闲任务时进入低功耗模式以
降低系统功耗。


## 11.2 FreeRTOS 睡眠模式

FreeRTOS 支持三种睡眠模式：

1. eAbortSleep - 该模式表示有任务已就绪、已挂起上下文切换，或在调度器挂起期间
	已发生 tick 中断但被挂起。它用于通知 RTOS 放弃进入睡眠模式。

2. eStandardSleep - 允许进入不超过预计空闲时间的睡眠模式。

3. eNoTasksWaitingTimeout - 当没有任务在等待超时，安全进入只能由外部中断或复位
	唤醒的睡眠模式。
  
  
## 11.3 内置 Tickless Idle 功能与启用方式

对于支持该特性的移植，可在 FreeRTOSConfig.h 中将 `configUSE_TICKLESS_IDLE` 设为 1
以启用内置 Tickless Idle 功能。对于任意 FreeRTOS 移植（包括带内置实现的移植），
也可将 `configUSE_TICKLESS_IDLE` 设为 2 来使用用户自定义的 tickless idle 功能。

当启用 tickless idle 功能时，内核在满足以下两个条件时调用
`portSUPPRESS_TICKS_AND_SLEEP()` 宏：

1. 空闲任务是唯一可运行的任务，因为所有应用任务都处于 *Blocked* 或 *Suspended* 状态。

2. 在内核需要将某任务从 *Blocked* 状态切换为 *Ready* 状态之前，还有至少 n 个完整 tick
	周期将过去，其中 n 由 FreeRTOSConfig.h 中 `configEXPECTED_IDLE_TIME_BEFORE_SLEEP`
	设置。


### 11.3.1 portSUPPRESS_TICKS_AND_SLEEP() 宏

```c
portSUPPRESS_TICKS_AND_SLEEP( xExpectedIdleTime )
```
***清单 11.1*** *portSUPPRESS_TICKS_AND_SLEEP 宏原型*

`portSUPPRESS_TICKS_AND_SLEEP()` 中 `xExpectedIdleTime` 参数值等于任务下次将被

### 11.3.3 eTaskConfirmSleepModeStatus 函数

API `eTaskConfirmSleepModeStatus` 返回睡眠模式状态，用于判断是否可以进入睡眠以及
是否可以无限期睡眠。该功能仅在 `configUSE_TICKLESS_IDLE` 设为 1 时可用。

<a name="list11.3" title="Listing 11.3 The eTaskConfirmSleepModeStatus API function prototype"></a>
 
```c
eSleepModeStatus eTaskConfirmSleepModeStatus( void );
```
***清单 11.3*** *eTaskConfirmSleepModeStatus API 函数原型*


若在 `portSUPPRESS_TICKS_AND_SLEEP()` 内调用 `eTaskConfirmSleepModeStatus()` 返回
`eNoTasksWaitingTimeout`，则 MCU 可无限期保持深度睡眠状态。
`eTaskConfirmSleepModeStatus()` 只有在以下条件全部满足时才会返回
`eNoTasksWaitingTimeout`：

+ 未使用软件定时器，因此调度器未来不需执行定时器回调函数。
  
+ 所有应用任务都处于 *Suspended* 状态，或处于 *Blocked* 状态且超时值为
  `portMAX_DELAY`，因此调度器未来不会在固定时间将任务切换为 *Ready* 状态。

为避免竞争条件，调用 `portSUPPRESS_TICKS_AND_SLEEP()` 前会挂起调度器，完成后再恢复。
这确保 MCU 退出低功耗状态与 `portSUPPRESS_TICKS_AND_SLEEP()` 执行完成之间，
应用任务不会执行。此外，`portSUPPRESS_TICKS_AND_SLEEP()` 需要在停止定时器与进入
睡眠模式之间建立一个短临界区，确保可继续进入睡眠模式。
`eTaskConfirmSleepModeStatus()` 应在该临界区内调用。

此外，FreeRTOS 在 FreeRTOSConfig.h 中提供了两个接口宏，允许应用在 MCU 进入
低功耗状态前与退出后分别添加额外处理步骤。

## 11.4 实现 portSUPPRESS_TICKS_AND_SLEEP() 宏

若所用 FreeRTOS 移植未提供 `portSUPPRESS_TICKS_AND_SLEEP()` 默认实现，
应用编写者可在 FreeRTOSConfig.h 中定义 `portSUPPRESS_TICKS_AND_SLEEP()` 实现。
若移植已提供默认实现，应用编写者也可在 FreeRTOSConfig.h 中重新定义该宏以覆盖
默认实现。

以下源码展示了应用编写者可能实现 `portSUPPRESS_TICKS_AND_SLEEP()` 的示例。
该示例较基础，会引入内核维护时间与真实时间之间的偏差。示例中调用的函数中，
只有 `vTaskStepTick()` 与 `eTaskConfirmSleepModeStatus()` 属于 FreeRTOS API。
其他函数依赖硬件时钟与低功耗模式，因此需由应用编写者提供。

<a name="list11.6" title="Listing 11.6 An example of a user defined implementation of portSUPPRESS_TICKS_AND_SLEEP()"></a>

```c
/* 首先定义 portSUPPRESS_TICKS_AND_SLEEP() 宏。参数是内核下一次需要执行前的
	时间（以 tick 为单位）。 */

#define portSUPPRESS_TICKS_AND_SLEEP( xIdleTime ) vApplicationSleep( xIdleTime )

/* 定义由 portSUPPRESS_TICKS_AND_SLEEP() 调用的函数。 */
void vApplicationSleep( TickType_t xExpectedIdleTime )
{
	 unsigned long ulLowPowerTimeBeforeSleep, ulLowPowerTimeAfterSleep;

	 eSleepModeStatus eSleepStatus;

	 /* 从在低功耗状态下仍可运行的时间源读取当前时间。 */
	 ulLowPowerTimeBeforeSleep = ulGetExternalTime();

	 /* 停止产生 tick 中断的定时器。 */
	 prvStopTickInterruptTimer();

	 /* 进入一个不会影响唤醒 MCU 的中断的临界区。 */
	 disable_interrupts();

	 /* 确保仍可进入睡眠模式。 */
	 eSleepStatus = eTaskConfirmSleepModeStatus();

	 if( eSleepStatus == eAbortSleep )
	 {
		  /* 自该宏执行以来已有任务离开 Blocked 状态，或有上下文切换被挂起。
			  不要进入睡眠状态。重启 tick 并退出临界区。 */
		  prvStartTickInterruptTimer();
		  enable_interrupts();
	 }
	 else
	 {
		  if( eSleepStatus == eNoTasksWaitingTimeout )
		  {
				/* 无需配置在固定时间将微控制器从低功耗状态唤醒的中断。 */
				prvSleep();
		  }
		  else
		  {
				/* 配置中断，在内核下一次需要执行时将微控制器从低功耗状态唤醒。
					该中断必须由在低功耗状态下仍可运行的源产生。 */
				vSetWakeTimeInterrupt( xExpectedIdleTime );

				/* 进入低功耗状态。 */
				prvSleep();

				/* 计算微控制器实际处于低功耗状态的时间。
					若微控制器由 vSetWakeTimeInterrupt() 配置以外的中断唤醒，
					该时间将小于 xExpectedIdleTime。
					注意：调用 portSUPPRESS_TICKS_AND_SLEEP() 前调度器已挂起，
					在其返回时恢复。因此在该函数完成前不会有其他任务执行。 */
				ulLowPowerTimeAfterSleep = ulGetExternalTime();

				/* 修正内核 tick 计数，以补偿微控制器在低功耗状态中的时间。 */
				vTaskStepTick( ulLowPowerTimeAfterSleep - ulLowPowerTimeBeforeSleep );
		  }

		  /* 退出临界区——可能可以在 prvSleep() 调用后立即执行。 */
		  enable_interrupts();

		  /* 重启产生 tick 中断的定时器。 */
		  prvStartTickInterruptTimer();
	 }
}
```
***清单 11.6*** *用户自定义 portSUPPRESS_TICKS_AND_SLEEP() 的示例实现*


## 11.5 空闲任务钩子函数

空闲任务可以选择性地调用应用定义的钩子（回调）函数——空闲钩子。空闲任务以最低
优先级运行，因此空闲钩子仅在没有更高优先级任务可运行时执行。这使空闲钩子成为
将处理器置于低功耗状态的理想位置——当系统无处理任务时自动省电。
只有在 FreeRTOSConfig.h 中将 `configUSE_IDLE_HOOK` 设为 1 时，空闲钩子才会被调用。

<a name="list11.7" title="Listing 11.7 The vApplicationIdleHook API function prototype"></a>

```c
void vApplicationIdleHook( void );
```
***清单 11.7*** *vApplicationIdleHook API 函数原型*


只要空闲任务运行，空闲钩子就会反复被调用。必须确保空闲钩子不调用任何可能阻塞的
API 函数。此外，如果应用使用 `vTaskDelete()` API，则空闲钩子必须允许周期性返回，
因为空闲任务负责清理被删除任务由 RTOS 内核分配的资源。
